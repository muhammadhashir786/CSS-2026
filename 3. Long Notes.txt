Selectors:

there are selectors in CSS
selectors are used to select HTML elements on which styles will be apply
Selectors indicate which element the rule applies to.

CSS selectors target or select HTML elements to apply styles on it
CSS selectors are case sensitive 

Declaration:

"{}" are written just after the "selector" which is called "Declaration Blocks" inside that we write CSS styles which is called "Declaration"
Declarations are split into two parts (a property and a value) and are separated by a colon ":"
semi-colon ";" is written at the end of declaration
we can specify several properties in one declaration, each separated by a semi-colon ";"

Property:

Properties indicate the aspects of the element you want to change. 
Example:    color, font, width, height and border.

Value:
 
Values specify the settings you want to use for the chosen properties.
Example:    property is color and its value is "red"

there are many selectors in CSS:

                                values

Cascade:

Cascade algorithm is used to resolve any conflicts in CSS
what if defines 2 or more styles for same HTML element, what will be apply?
it decides that which property will apply and which will not apply
here comes cascade rules like precedence, specificity, Priority

4 Rules of Cascade:

1. position and order of appearance 
2. specificity
3. origin - when developer did not define css - browser agent stylesheet, where css comes from  
4. !importance

1. Position:

if multiple styles define using same selector, the last style will apply

2. Specificity:

defines "Priority no" which elment is most specific and highest Priority

if multiple styles define using different selectors, then specificity rule comes where max score counts of selector:
the element which have max score, its style will apply

Selectors                                       Scores

Universal selector                              0
Element selector & pseudo element               1            
class, attribute selector & pseudo classes      10
id selector                                     100
inline css using style attribute                1000 
!important flag                                 10000

class specificity = 10
element + class selector = 1 + 10 = 11 
element + class + attribute selector = 1 + 10 + 10 = 21 
element + class + id selector = 1 + 10 + 100 = 111 

Note:
    try to use minimum specific selectors, so that we can over-write it easily 

3. origin:

browser agent stylesheet 
where css comes from,
if user did not define css 


Inheritence:

Every CSS Property have its "Value"
we define "value" of css "property" for all the html elements
what happens if no value is provided for any property of an element
the value is either "inherited" from the parent element or set to its "initial" value bydefault in css

Types of Properties:

there are 2 types of properties in css:

1. inherited property 
2. non-inherited property 

1. Inherited Property:

inherited properties take the "values" from their parent element bydefault, if no value is provided

Examples:
    color, font-family, font-size, line-height, and text-align, etc

CSS Code:

p {
  color: blue;
  font-size: 20px;
}

HTML Code:

<p>This is a paragraph with some <strong>important</strong> text.</p>

2. Non-inherited Property:

Non-inherited properties do not take the "values" from their parent element bydefault
if it is needed, we define manually by using "inherit" keyword in "value"  

Examples:
    border, background, margin, padding, width, and height

Inherit Keyword:

"inherit" keyword gives values to those css properties that value does not defined  
by using this keyword, we can manually set "inherited value" for any element
it works in both inherit or non-inherited properties
